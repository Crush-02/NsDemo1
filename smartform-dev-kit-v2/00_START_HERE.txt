# SmartForm AI开发资料包 v2.0 - 使用指南

## 改版说明

v1.0的问题：
- 文档驱动导致AI先写校验逻辑再适配Luckysheet，本末倒置
- Python参考代码干扰了AI的TypeScript实现思路
- PRD过长(4万字)导致AI注意力分散
- 三阶段划分不合理，Step1需要Step2的文档

v2.0的改进：
- **功能驱动**：先搭框架再写校验，符合实际开发流程
- **纯前端语境**：去除所有Python代码，仅保留规则JSON
- **精简文档**：每个Step控制在3000字以内，聚焦当前任务
- **自包含**：每个Step的资料独立完整，不依赖其他Step

## 三步开发流程

### Step 1: 搭建Luckysheet框架（目标：可编辑的Excel体验）

**做什么**：
- 初始化 Vite + Vue3 + TypeScript 项目
- 集成 Luckysheet，实现基本的表格渲染和编辑
- 实现Excel导入/导出
- 实现自动保存

**成功标准**：
- [ ] 页面能加载显示25列的物业导入模板表头
- [ ] 可以在单元格中输入和编辑数据
- [ ] Tab/Enter键能切换单元格
- [ ] 能导入一个.xlsx文件并显示数据
- [ ] 能将表格数据导出为.xlsx文件

**发送给AI**：`step1_framework/` 下的2个文件

---

### Step 2: 实现简单校验（目标：基础规则能跑通）

**做什么**：
- 在Step1的框架上，接入校验引擎
- 实现无条件触发的简单规则（必填、格式）
- 实现基本的错误UI反馈（红色边框+Tooltip）

**成功标准**：
- [ ] 房产简称为空时显示红色边框
- [ ] 楼层输入"-1"时显示橙色警告"必须为正整数"
- [ ] 日期格式不是YYYY-MM-DD时提示错误
- [ ] 错误信息以Tooltip形式展示

**发送给AI**：`step2_simple_validation/` 下的3个文件

---

### Step 3: 完善所有校验（目标：20条规则全部生效）

**做什么**：
- 实现条件依赖规则（有业主信息→售楼日期+业主类型必填）
- 实现跨行校验（唯一性、一致性）
- 实现三级校验机制（输入时/失焦时/保存前）
- 完善UI交互（条件依赖可视化、校验面板）

**成功标准**：
- [ ] 填写业主名称后，售楼日期和业主类型自动标记为待填写
- [ ] 房产简称重复时高亮提示
- [ ] 导出前执行全量校验，有错误则拦截
- [ ] 校验面板显示错误统计和列表

**发送给AI**：`step3_full_validation/` 下的2个文件

## 使用原则

1. **每步完成后验证再进入下一步** - 不要跳步
2. **只发送当前Step的文件** - 不要提前发后续Step的资料
3. **如果AI偏离方向，立即纠正** - 不要等它写完再改
4. **每步的代码必须在浏览器中可运行** - 不接受"伪代码"
