## 1. 工具概述

新视窗填表工具是一款面向物业管理的**在线数据填表与校验工具**，主要解决房产信息、应收已收数据的批量录入、格式校验和导出问题。

### 核心特性

| 特性 | 说明 |
|------|------|
| Excel批量导入 | 支持 `.xlsx` / `.xls` 文件导入，自动匹配字段 |
| 智能字段映射 | Excel表头与工具字段精确匹配，未匹配字段给出提示 |
| 实时数据校验 | 编辑单元格时即时校验格式和必填规则 |
| 跨行一致性校验 | 检测房产简称唯一性、证件号码一致性等跨行规则 |
| 条件依赖校验 | 根据已填内容自动判断其他字段是否变为必填 |
| 校验面板 | 侧边栏展示所有校验问题，点击可定位到对应单元格 |
| 自动保存 | 每30秒自动保存到浏览器本地，刷新页面可恢复 |
| Excel导出 | 校验通过后导出标准格式Excel文件，文件名含项目名称 |
| 大数据量支持 | 1500+行数据导入不卡顿，分步后台校验 |

## 2. 快速开始

### 2.1 环境要求

- 浏览器：Chrome 90+、Edge 90+、Firefox 90+（推荐Chrome）
- 屏幕分辨率：1920×1080 及以上
- 无需安装任何软件，浏览器直接访问即可使用

### 2.2 启动步骤

1. 打开浏览器，访问工具地址
2. 进入首页，选择需要使用的模板（房产信息 / 应收已收）
3. 进入编辑器界面，开始录入数据或导入Excel

### 2.3 基本操作流程

```
选择模板 → 导入Excel/手动录入 → 等待自动校验 → 查看校验结果 → 修正错误 → 导出Excel
```

## 5. 数据导入

### 5.1 导入Excel文件

1. 点击工具栏的 **「导入Excel」** 按钮
2. 在弹出的文件选择器中，选择 `.xlsx` 或 `.xls` 格式的Excel文件
3. 工具自动读取Excel数据并进行字段匹配

### 5.2 字段匹配规则

工具采用**精确匹配**方式将Excel表头与工具字段对应：

- Excel表头文字与工具字段名称**完全一致**时自动匹配
- 未匹配的Excel字段会弹出提示："以下Excel字段未识别，已跳过：xxx"
- 工具中存在但Excel中缺失的字段会提示："以下字段Excel中缺失：xxx"
- 如果没有任何字段匹配，导入终止并提示错误

> **提示**：请确保Excel的表头行（第一行）字段名称与工具字段名称完全一致，包括空格和标点符号。

### 5.3 导入后校验

导入完成后，工具自动启动**分步校验**：

1. 界面弹出遮罩层，显示"数据校验进行中，请稍等"
2. 进度条实时显示校验进度
3. 校验期间不可编辑表格
4. 校验完成后遮罩层自动消失，校验结果展示在校验面板中

## 6. 数据校验

### 6.1 校验机制说明

工具采用**三层校验架构**，确保数据质量：

| 层级 | 触发时机 | 说明 |
|------|---------|------|
| **即时校验** | 编辑单元格时 | 立即校验当前单元格的格式和必填规则 |
| **失焦校验** | 编辑后点击其他单元格时 | 校验当前行的所有简单规则和条件规则 |
| **跨行校验** | 失焦后800ms | 校验涉及多行的规则（唯一性、一致性等） |

#### 校验优先级

| 严重度 | 标签 | 说明 |
|--------|------|------|
| CRITICAL | 错误 | 必须修正，否则无法导出 |
| HIGH | 警告 | 建议修正，可强制导出 |
| MEDIUM | 提示 | 建议关注 |

### 6.2 校验规则详解

#### 6.2.1 必填规则

以下字段在任何情况下都必须填写：

| 规则 | 字段 | 错误信息 |
|------|------|---------|
| 项目名称_必填 | 项目名称 | 项目名称不可为空 |
| 楼栋名称_必填 | 楼栋名称 | 楼栋名称不可为空 |
| 楼层_必填 | 楼层 | 楼层不可为空且必须为正整数 |
| 房号_必填 | 房号 | 房号不可为空 |
| 房产简称_必填 | 房产简称 | 房产简称不可为空且在项目中唯一 |
| 房产类型_必填 | 房产类型 | 房产类型不可为空，请选择枚举值 |

#### 6.2.2 格式规则

| 规则 | 字段 | 校验逻辑 |
|------|------|---------|
| 格式_计费面积 | 计费面积 | 有值时必须为正数，最多4位小数 |
| 格式_日期 | 售楼/收房/入住/出租日期 | 有值时必须为YYYY-MM-DD格式，且为真实日期 |
| 格式_客户类型 | 业主/租户客户类型 | 有值时必须为"个人"或"企业" |
| 格式_房产类型 | 房产类型 | 必须为：机动车位、非机动车位、商品房、写字楼、商铺、别墅 |
| 格式_业主证件类型 | 业主证件类型 | 有值时必须为"身份证"或"其他" |
| 格式_业主证件号码 | 业主证件号码 | 证件类型=身份证时，需符合GB 11643-1999标准（18位+出生日期+校验码） |
| 格式_证件号码 | 租户证件号码 | 有值时需符合18位身份证格式 |

#### 6.2.3 条件必填规则

当某些字段已填写时，其他字段自动变为必填：

| 触发条件 | 必填字段 | 错误信息 |
|---------|---------|---------|
| 有业主信息（名称/类型/电话任一有值） | 售楼日期 | 业主客户信息填写时，售楼时间必填 |
| 有业主信息 | 业主客户类型 | 业主的客户信息填写时，业主客户类型必填 |
| 收房日期有值 | 售楼日期 | 收房日期填写时，售楼日期必填 |
| 入住日期有值 | 售楼日期 | 入住日期填写时，售楼日期必填 |
| 业主客户类型=企业 | 企业联系人 | 业主客户类型为企业时，企业联系人必填 |
| 有租户信息 | 出租开始日期 | 租户的信息填写时，出租日期必填 |
| 有租户信息 | 出租结束日期 | 租户的信息填写时，出租结束日期必填 |
| 出租日期有值 | 租户客户名称 | 出租日期填写时，租户客户名称必填 |
| 租户联系电话有值 | 租户客户名称 | 租户联系电话填写时，租户客户名称必填 |
| 租户客户类型=企业 | 租户企业联系人 | 租户客户类型为企业时，租户企业联系人必填 |
| 业主证件号码有值 | 业主证件类型 | 填写业主证件号码时，业主证件类型必填 |

#### 6.2.4 跨行校验规则

跨行校验检查多行数据之间的关系：

| 规则 | 校验逻辑 | 错误信息 |
|------|---------|---------|
| 房产简称_唯一性 | 同一项目中房产简称不能重复 | 房产简称"xxx"重复出现N次，在项目中必须唯一 |
| 日期顺序 | 收房日期不能小于售楼日期 | 收房日期不能小于售楼日期 |
| 客户信息一致性 | 同一租户证件号码对应不同的租户名称 | 存在相同证件号码但是名称不同的客户，请核对客户信息 |
| 业主证件号码_一致性 | 不同业主（名称+电话不同）不能使用相同的证件号码 | 业主证件号码"xxx"被不同业主使用，不同业主的证件号码不能相同 |

### 6.3 校验面板使用

#### 打开/关闭面板

- 点击工具栏的 **「校验面板」** 按钮打开
- 点击面板右上角的关闭按钮或再次点击 **「隐藏面板」** 关闭

#### 面板内容

校验面板包含两个标签页：

**校验问题** 标签页：
- 按行分组展示所有校验错误和警告
- 每条错误显示：严重度标签、字段名、错误信息
- 点击错误条目，表格自动滚动到对应单元格并居中显示
- 错误单元格会以红色背景标记

**待填写** 标签页：
- 展示因条件触发而变为必填但尚未填写的字段
- 每条显示行号、字段名和触发原因
- 点击条目可跳转到对应单元格

#### 统计概览

面板顶部显示三个统计数字：
- **错误**（红色）：CRITICAL级别的问题数量
- **警告**（橙色）：HIGH/MEDIUM级别的问题数量
- **待填写**（绿色）：条件必填但未填写的字段数量

### 6.4 校验状态栏

界面底部的状态栏实时显示：

- **已填写行数**：如"已填写 50/100 行"
- **校验进度**：导入后校验时显示进度条和百分比
- **错误/警告数量**：如"3个错误"、"5个警告"
- **无校验问题**：所有校验通过时显示

### 6.5 重新校验

如需手动触发全量校验：

1. 打开校验面板
2. 点击底部的 **「重新校验」** 按钮
3. 等待校验完成

> **提示**：日常编辑时无需手动重新校验，工具会在编辑后自动触发增量校验。

## 7. 数据导出

### 7.1 导出步骤

1. 点击工具栏的 **「导出Excel」** 按钮
2. 工具自动执行全量校验
3. 根据校验结果：
   - **有错误（CRITICAL）**：弹出对话框列出错误，**无法导出**，需修正后重试
   - **仅有警告（HIGH/MEDIUM）**：弹出确认框，可选择继续导出或取消
   - **无任何问题**：直接导出

### 7.2 导出文件名

导出文件名格式为：`{项目名称}房产信息数据_{日期}_{时间}.xlsx`

示例：`爱琴广场房产信息数据_20260621_134701.xlsx`

项目名称取自表格中第一个非空的项目名称单元格。

### 7.3 导出内容

- 第一行为表头（32个字段名）
- 后续行为数据行（跳过全空行）
- 列宽按工具预设宽度设置

## 8. 自动保存与草稿恢复

### 8.1 自动保存

- 工具每 **30秒** 自动将当前表格数据保存到浏览器本地（localStorage）
- 保存内容包括所有单元格的值和格式
- 无需手动操作，完全自动进行

### 8.2 草稿恢复

当重新打开工具时：

1. 如果检测到本地有草稿数据，弹出提示框
2. 提示框显示草稿保存的时间，如"检测到 2026-06-21 13:47:01 保存的草稿数据，是否恢复？"
3. 点击 **「恢复」** 恢复草稿数据，点击 **「忽略」** 放弃草稿

### 8.3 数据安全说明

- 所有数据仅存储在**当前浏览器**的localStorage中
- 数据不会上传到任何服务器
- 清除浏览器缓存会导致草稿数据丢失
- 不同浏览器/设备之间数据不互通
- 关闭浏览器标签页不会丢失数据（自动保存已保存）

## 10. 常见问题

### Q1：导入Excel后提示"以下Excel字段未识别"

**原因**：Excel中的表头名称与工具字段名称不完全一致。

**解决**：检查Excel表头，确保与工具字段名称完全一致（包括空格、标点）。可参考本文第9章的字段名称表。

### Q2：导入Excel后提示"以下字段Excel中缺失"

**原因**：Excel中缺少工具定义的某些字段列。

**解决**：这是信息提示，不影响导入。缺失的字段在表格中保持为空，如果是必填字段，校验时会报错。

### Q3：编辑单元格后没有立即出现校验提示

**原因**：即时校验只检查当前单元格的格式和必填规则，条件规则和跨行规则在失焦后触发。

**解决**：点击其他单元格（触发失焦），等待约1秒后校验结果会自动出现。

### Q4：校验面板显示"待填写"但该字段不是必填的

**原因**：该字段因条件触发变为必填。例如，填写了业主信息后，售楼日期和业主客户类型自动变为必填。

**解决**：查看待填写项的描述信息，了解触发原因，补充填写即可。

### Q5：导出时提示"存在N个错误，请修正后再导出"

**原因**：存在CRITICAL级别的校验错误，不允许导出。

**解决**：打开校验面板，逐条修正所有错误后重新导出。

### Q6：刷新页面后数据丢失

**原因**：工具每30秒自动保存一次，如果上次编辑后未满30秒就刷新，可能丢失最近的数据。

**解决**：重新打开页面时，工具会提示恢复草稿。如果未出现提示，说明数据未能保存。

### Q7：导入大数据量（1500+行）时页面卡顿

**原因**：工具已针对大数据量做了分步校验优化，但浏览器内存有限。

**解决**：
- 导入后等待校验完成再操作
- 校验期间不要频繁滚动或编辑
- 如持续卡顿，尝试关闭其他浏览器标签页释放内存

### Q8：业主证件号码填了"其他"类型的号码，校验提示格式错误

**原因**：业主证件类型选择"其他"时，不对证件号码做格式校验；但选择"身份证"时需符合GB 11643-1999标准。

**解决**：确认证件类型选择正确。如果是非身份证件，请选择"其他"。

### Q9：两条不同业主的记录使用了相同证件号码，校验没有报错

**原因**：此为跨行校验规则，需要在失焦后约1秒才会触发。

**解决**：编辑完证件号码后点击其他单元格，等待1秒后校验面板会显示错误。如仍未显示，点击校验面板的"重新校验"按钮。

### Q10：表头行被冻结，无法滚动

**说明**：这是正常设计。表头行（第一行）默认冻结，滚动时表头始终可见，方便对照字段名称。这是"冻结首行"功能，不是bug。

### 高效录入建议

1. **先导入后修改**：优先使用Excel导入批量数据，再在工具中逐条修正校验错误
2. **按顺序填写**：先填房产基本信息，再填业主信息，最后填租户信息，避免条件必填报错
3. **善用校验面板**：导入后打开校验面板，按行逐条修正错误
4. **定期导出备份**：修正部分错误后可先导出一份，避免意外丢失
5. **必填字段识别**：表头中文字为**红色**的列为必填字段

### Excel模板准备建议

1. 确保Excel第一行为表头，字段名称与工具完全一致
2. 日期字段统一使用 `YYYY-MM-DD` 格式
3. 枚举字段（房产类型、客户类型等）使用工具规定的值
4. 删除Excel中工具不存在的多余列（会被跳过但不影响导入）
5. 确保没有合并单元格
