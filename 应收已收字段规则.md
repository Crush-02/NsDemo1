# 应收已收导入字段规则

## 一、模板结构概览

共 **18 列**，分为三大区域：**基本信息**、**应收**、**已收**。

| 区域 | 包含字段 |
|---|---|
| 基本信息（A-C列） | 项目名称、房产简称、收费对象 |
| 应收（D-M列） | 收费科目、收费标准、计费开始日期、计费结束日期、应收日期、应收金额、减免金额、减免分类、减免日期、应收备注 |
| 已收（N-R列） | 缴款金额、缴款日期、支付方式、已收备注、合同编号 |

---

## 二、字段格式要求

| 字段 | 格式要求 |
|---|---|
| 计费开始日期 | 短日期格式，如 2026/6/4 |
| 计费结束日期 | 短日期格式，如 2026/6/4 |
| 应收日期 | 短日期格式，如 2026/6/4 |
| 应收金额 | 数值格式，最多保留 2 位小数 |
| 减免金额 | 数值格式，最多保留 2 位小数 |
| 减免分类 | 下拉选择框 |
| 减免日期 | 短日期格式，如 2026/6/4 |
| 缴款金额 | 数值格式，最多保留 2 位小数 |
| 缴款日期 | 短日期格式，如 2026/6/4 |
| 支付方式 | 下拉选择 |

### 支付方式可选值

现金、银行转账、微信、支付宝、对公转账

### 减免分类可选值

空置减免、特殊减免、活动优惠减免

### 导入文件格式

仅支持 **.xlsx** 格式，单次导入最多 **1,500 行** 数据。

---

## 三、必填字段（红色表头）

表头为红色字体的字段为必填项，共 **7 个**：

| 区域 | 必填字段 |
|---|---|
| 基本信息 | 项目名称、房产简称、收费对象 |
| 应收 | 收费科目、计费开始日期、计费结束日期、应收金额 |

> 其余字段（收费标准、应收日期、减免金额、减免分类、减免日期、应收备注、缴款金额、缴款日期、支付方式、已收备注、合同编号）为非必填项。

---

## 四、数值校验规则

### 金额字段校验

| 字段 | 必填性 | 校验规则 |
|---|---|---|
| 应收金额 | 必填 | 必须大于 0 |
| 减免金额 | 非必填 | 如填写则必须大于 0，不可填 0 或负数 |
| 缴款金额 | 非必填 | 如填写则必须 ≤ 应收金额 |

---

## 五、导入规则

### 规则 1：项目名称一致性

一个表格内项目名称必须保持一致，禁止在一个表格中出现多个不同的项目名称。

### 规则 2：房产简称可重复

房产简称在同一表格内不要求唯一，允许重复。（重复校验由规则8中的组合条件控制）

### 规则 3：减免金额与减免分类联动（一）

当减免金额列填入数据时，减免分类为必填项。

### 规则 4：减免金额与减免分类联动（二）

当减免分类列填入数据时，减免金额为必填项。

### 规则 5：缴款三字段联动

缴款金额、缴款日期、支付方式三个字段本身为非必填项，但只要其中任一字段填入数据，则其余两个字段也必须填写。

即：三者要么全部为空，要么全部填入数据。

### 规则 6：缴款金额上限

缴款金额如有填写，则必须小于或等于对应的应收金额（小于表示部分缴款，等于表示全部缴款）。

### 规则 7：计费日期逻辑

计费结束日期必须晚于计费开始日期，日期区间跨度无时间限制。

### 规则 8：数据重复校验

两条数据的以下四个字段不可全部相同，否则导入时会报重复：

**房产简称 + 收费科目 + 计费开始日期 + 计费结束日期**

即：同一房产的同一收费科目在同一计费周期内只能有一条数据。

---

## 六、其他说明

- 房产简称必须与房产导入表中的简称保持一致，否则无法导入。
- 收费对象为房产导入表中的业主/租户姓名。
- 同一个房产或住户可以有多行数据，需分行导入（如同时欠物业费和水费）。
- 各项目收费科目命名规则应保持一致，禁止出现同一科目不同项目命名不一致的情况（如 A 项目叫"物业费"，B 项目叫"物业服务费"）。
- 收费标准需加上单位（如 2.5元/平方/月），无格式限制，随意填写即可。
- 应收账期由系统根据应收日期的年份和月份自动生成（例如应收日期为 2023/03/08，则应收账期为 202303）。
- 备注字段（应收备注、已收备注）无字数限制或特殊要求。
- 减免日期为非必填项。
- 合同编号为非必填项。
